Java 富 Internet 应用程序决策指南

applet 和 Java Web Start 应用程序都被视为 Rich Internet Applications (RIAs) (富 Internet 应用程序)。RIA 会提示用户在第一次启动时运行权限，并且应使用来自受信任的证书颁发机构的证书进行签名。评估 applet 和 Java Web Start 应用程序的以下特征，以决定如何部署 RIA。

Applets

  • Applet 在浏览器的上下文中运行。
  • Applet 可以访问会话 cookie 和持久性 cookie。
  • Applet 可以与嵌入它们的网页进行交互。Applet 可以遍历和操作网页的文档对象模型，并与网页中的 JavaScript 进行交互。JavaScript 代码可以访问 applet 的公共方法和变量。
  • 可以使用 Java 网络启动协议(JNLP)启动 Applet。使用 JNLP 启动时，允许沙箱 applet 访问持久存储，下载控制，文件 I/O 等。使用 JNLP 启动的 Applet 具有与 Java Web Start 应用程序相当的功能。
    Applet 也可以在没有 JNLP 的情况下启动，并且可以限制在安全沙箱中或请求所有权限。
  • Applet 可以请求特定版本的 Java Runtime Environment 软件来执行。

Java Web Start 应用程序

  • Java Web Start 应用程序第一次从网页启动。随后，可以从网页或桌面快捷方式重新启动它们。
  • Java Web Start 应用程序不在浏览器的上下文中运行。应用程序无法与网页中的 HTML 和 JavaScript 交互，只能访问持久性 cookie。
  • 允许 Java Web Start 应用程序访问持久存储，文件 I/O 和其他客户端相关服务。
  • Java Web Start 应用程序可以请求特定版本的 Java Runtime Environment 软件以供执行。
